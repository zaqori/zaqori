# ZAQORI Admin Panel — Setup Guide

Your ZAQORI site ships with a **Decap CMS** admin panel at `https://zaqori.com/admin/`.
This lets you create, edit, and publish blog articles through a web interface — no
code required.

## How It Works

```
┌─────────────┐     ┌──────────────────┐     ┌─────────────┐     ┌──────────────┐
│ You log in  │ ──▶ │ Decap CMS        │ ──▶ │ Netlify     │ ──▶ │ New article  │
│ via /admin/ │     │ commits to Git   │     │ redeploys   │     │ is live      │
└─────────────┘     └──────────────────┘     └─────────────┘     └──────────────┘
```

1. You log in at `https://zaqori.com/admin/` with email + password
2. You create or edit a blog article in the CMS
3. Decap CMS saves it as a markdown file in `blog/content/` and pushes to GitHub
4. Netlify auto-rebuilds the site and the new article appears

## One-Time Setup (5 minutes)

### Step 1: Enable Netlify Identity

1. Go to your **Netlify Dashboard** → select your `zaqori.com` site
2. Click **Identity** in the top nav
3. Click **Enable Identity**
4. Under **Registration**, select **Invite only** (so only you can register)
5. Under **Services** → **Git Gateway**, click **Enable Git Gateway**
6. Authorize Netlify to access your GitHub repo

### Step 2: Invite Yourself as an Admin

1. In Netlify Identity, click **Invite users**
2. Enter your email (`zaqori.official@gmail.com`)
3. Check **Admin** under Roles
4. Click **Send invite**
5. Check your inbox → click the invite link → set your password

### Step 3: Configure the Build Hook (Optional but Recommended)

Decap CMS saves articles as **markdown files** in `blog/content/`. Your site serves
**HTML files**. To convert markdown → HTML automatically:

1. In Netlify → **Site settings** → **Build & deploy** → **Build settings**
2. Set **Build command** to: `python3 build.py`
3. Set **Publish directory** to: `.` (dot — files at root, no wrapper folder)
4. Click **Save**

Now every time you publish through the CMS, Netlify will:
1. Pull the new markdown file from GitHub
2. Run `build.py` to convert it to a proper HTML article
3. Update `blog/index.html` with the new article card
4. Update `sitemap.xml`
5. Deploy the updated site

> **No Python available on Netlify?** Skip this step — the article will still be saved
> to GitHub, and you can run `python3 build.py` locally then push the HTML.

## Using the Admin Panel

### Logging In
1. Visit `https://zaqori.com/admin/`
2. Click **Login with Netlify Identity**
3. Enter your email + password

### Creating a New Article
1. Click **New Blog Article** in the left sidebar
2. Fill in the fields:
   - **Title** — the article headline
   - **URL Slug** — e.g. `how-to-save-money` (lowercase, hyphens, no spaces)
   - **Publish Date** — when it should appear
   - **Category** — pick from the dropdown
   - **Featured Image** — upload a 1200×630px image
   - **Meta Title (SEO)** — what Google shows in search results
   - **Meta Description (SEO)** — 150-160 char summary for search results
   - **Article Body** — write in Markdown (supports headings, bold, lists, images, links)
3. Toggle **Draft** to OFF when ready to publish
4. Click **Publish** → then **Publish now** in the top right

### Editing an Existing Article
1. Click **Blog Articles** in the left sidebar
2. Click the article you want to edit
3. Make your changes
4. Click **Publish** → **Publish now**

### Uploading Images
1. In the article editor, click **Choose an image** under Featured Image
2. Drag & drop or browse to upload
3. Images are saved to `images/articles/`

## Troubleshooting

### "Site can't be reached" at /admin/
- Make sure the `admin/` folder is in your deployed site
- Check the Netlify deploy log for any errors
- Try `https://zaqori.com/admin/index.html` directly

### "Login failed"
- Confirm Netlify Identity is enabled (Step 1 above)
- Confirm you accepted the invite email and set a password
- Check that Git Gateway is enabled

### New article doesn't appear on the site
- If you set a build command: check the Netlify deploy log for `build.py` output
- If you didn't set a build command: run `python3 build.py` locally, then push the generated HTML files

### Image upload fails
- Check that the `images/articles/` folder exists in your repo
- Verify the GitHub repo permissions in Netlify → Identity → Services → Git Gateway

## Security Notes

- **Invite-only registration** is strongly recommended (set in Netlify Identity)
- Only users you explicitly invite can access `/admin/`
- All changes are tracked in Git history — you can always roll back
- The admin panel URL is `noindex, nofollow` (hidden from search engines)

## Customization

### Change the admin panel logo
Edit `admin/config.yml` → `logo_url` field

### Add new fields to articles
Edit `admin/config.yml` → `collections[0].fields` array, then update `build.py` to render them

### Add a new collection (e.g., Simulators)
Add a new entry under `collections:` in `admin/config.yml`

For more info: https://decapcms.org/docs/
